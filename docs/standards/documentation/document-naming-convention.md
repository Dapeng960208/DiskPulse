# 文档命名与链接

- 文件名和目录名使用英文 `kebab-case`，正文使用简体中文。
- 功能专题目录使用 `docs/features/<领域>/<功能>/`；文档文件名表达该功能内的职责，例如 `overview.md`、`backend.md`、`frontend.md`、`operations-guide.md`。
- 使用语义明确的名称；不使用 `new.md`、`temp.md`、`final-v2.md` 等临时名称。
- 文档内部链接使用相对路径。链接到同一专题的内容时优先链接到其事实来源，而不是复制段落。
- 移动或删除文件前，全仓搜索旧路径并更新引用；完成后检查所有 Markdown 链接。

目录职责、分类边界和更新时机见[文档规范](./documentation-standard.md)。
