# AI 流式异常未写入服务端日志

## 错误内容

`ai_chat_service.stream_message()` 对未归类异常统一发送安全的 SSE `error` 事件并将审计摘要写为“AI 服务暂不可用”，但此前没有服务端异常日志。运维人员只能看到前端失败提示和通用审计摘要，无法通过 Trace ID 定位真实异常栈。

## 解决方案

在流式失败边界记录 `trace_id`、审计 ID、会话 ID、用户 ID 和异常类型，并携带异常栈；继续向客户端和持久化审计仅返回安全提示，不记录提问、凭据或原始 Provider 载荷。

## 备注

- 分类：`backend`
- 出现次数：1
- 首次与最近出现：2026-07-20，`2026-07-20-ai-stream-failure-logging` 会话。
- 触发特征：同一模型的连续失败审计只包含“AI 服务暂不可用”，且没有工具调用轨迹或对应服务端异常。
