# 批量平移事件时间桶触发即时唯一键冲突

## 错误内容

Alembic 批量把性能事件关联桶减 8 小时时，PostgreSQL 抛出 `UniqueViolation uq_incident_correlation_bucket`。最终目标桶集合没有重复，但唯一约束是即时检查的；平移链中前一行先占用了后一行尚未迁出的旧桶，产生瞬时冲突并使事务回滚。

## 解决方案

先把所有受影响事件写入按事件 ID 唯一的临时时间桶，再从证据边界计算最终 30 分钟桶。迁移前额外模拟最终桶并检查重复组；最终更新仍放在同一事务中，任何真实冲突都会完整回滚。
