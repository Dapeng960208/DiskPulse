# AI 工具编排重复请求已成功结果

## 错误内容

在同一 AI 对话回合中，模型重复请求同一工具和相同参数时，服务端仍会再次调用业务接口；工具请求失败时也没有明确的改参重试指令。结果是已成功或为空的查询不能被稳定用于回答，工具轮次可能被无效消耗。

## 解决方案

在 `ai_chat_service` 以内存缓存键“工具名 + 规范化参数”复用同回合成功结果，并将轨迹标记为 `reused`；检测到重复后，下一次 Provider 调用禁用工具以强制基于已有结果作答。失败结果不缓存，服务端向模型追加安全的改参提示，连续三次失败才进入基于已有结果的无工具降级总结。用回归测试锁定性能/事件工具注册、结果复用、无工具总结和失败后改参行为。
