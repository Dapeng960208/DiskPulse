# QuestDB 重建 SQL 未引用保留字列名

## 错误内容

QuestDB 修复完成前七张表后，在创建 `user_storage_usages` 修复表时因列名 `limit` 报错：SQL 关键字作为表名或列名必须使用双引号。第八表未创建，第九表尚未执行。

## 解决方案

构建 QuestDB DDL 和复制 SQL 时，对生产表中的保留字列名使用经过白名单校验的双引号标识符。中断后先审计已完成表和备份，再通过重复 `--table` 参数只续跑剩余表，避免对已平移表再次减 8 小时。
