# 项目实时使用页签未传递完整内容高度

## 错误内容

项目详情“项目使用实时”页签直接渲染实时组件时，没有独立的全高 flex 根节点。异步页签、内容区滚动或父级高度重新计算后，趋势图和告警表可能按内容自身高度收缩，导致内容区下方出现大片空白。

## 解决方案

为 `ProjectDiskUsage.vue` 添加全高、可收缩的 flex 根节点，并让 `RealTimePage` 作为其 `flex: 1` 子项。通用实时组件的 `fillContent` 模式由此获得稳定的父级高度，工作区可以占满筛选和概要信息后的剩余内容区。

## 备注

- 首次出现：2026-07-20，`2026-07-20-project-realtime-fill-content` 会话。
- 最近出现：2026-07-20，出现次数：1。
- 差异：仅修复项目实时使用页签的高度传递，不改变趋势查询、容量单位或告警数据契约。
