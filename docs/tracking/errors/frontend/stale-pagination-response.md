# 并发分页旧响应覆盖当前页

## 错误内容

连续切换分页时，较早请求晚完成会覆盖最新页的数据与总数；更早失败最后完成时还会清空结果、写入错误并提前结束加载状态。

## 解决方案

每个分页页面维护递增请求序号。成功、失败和 `finally` 只有在序号仍为最新时才能更新数据、总数、错误和加载状态；旧网络请求可以完成，但结果被忽略。
