# 事件中心按编辑时间排序且技术关联上下文缺失

## 错误内容

事件中心列表使用 Incident 的 `updated_at` 排序，手工更新状态、严重度或认领信息会使较早发生的事件排在最近事件之前。存储集群的“最近证据”列直接展示 RFC 3339 字符串；详情证据的可展开区域只显示内部 `source_ref`，无法从界面直接判断其关联对象、范围和证据类型。

## 解决方案

列表查询改按 `last_evidence_at`、`opened_at` 和 ID 倒序，在数据库分页前完成排序；前向迁移将相关索引从 `updated_at` 替换为最新证据排序键。时间线改为按发生时间倒序。两个列表以本地可读时间展示最近证据。详情的“技术关联信息”保留可复制的原始标识，并同时标明关联对象、证据范围和证据类型；前端不解析 `source_ref` 的内部格式。
