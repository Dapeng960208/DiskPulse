# 四维容量耗尽风险交付记录

## 会话范围

- 将容量预测扩展为存储集群、项目、项目组和用户目录四个维度。
- 移除独立容量预测栏目，将用户展示收敛到资源详情的“耗尽风险”。
- 固定内置基线、风险分级、AI 输出与候选启用标准，并在管理页提供清晰提示。
- 同步功能事实文档、关联专题和测试契约。

## 已完成

- 后台容量任务为项目生成独立预测目标，并让四个维度共同参与 AI 候选生成和三窗口评估。
- 新增四维轻量风险接口；存储集群仅超级管理员可读，其他维度按项目 `reader` 和全局发布开关授权。
- 候选创建、启用和每日运行均要求关联 AI 模型仍为已配置且启用状态。
- 四个资源详情接入共享“耗尽风险”面板；根导航和列表操作移除独立入口，旧深链保留兼容重定向。
- 预测治理页展示内置基线、风险分级和 AI 候选启用标准，并只列出已启用模型。
- Mock 运行时和前后端测试契约已迁移到四维轻量风险。
- 新建容量预测 AI 专题，并同步事件中心、存储趋势、存储集群、项目 RBAC、产品能力和文档索引。

## 验证

- 后端：`74 passed`，覆盖四维风险、预测治理和事件中心相关测试。
- 前端：`60 passed`，覆盖容量预测、治理、详情接入、路由、API 和 Mock 相关测试。
- 前端受影响文件 ESLint 通过。
- Python 受影响文件语法检查通过。
- `pnpm run build:prod` 通过；保留既有 Vite 环境变量和大 chunk 警告。
- Browser Mock 验收通过：超级管理员治理页展示三组标准，候选对话框只呈现当前已启用模型；存储集群、项目、项目组和用户目录详情均可打开“耗尽风险”并只显示风险、预计耗尽和判断依据。
- Browser 权限验收通过：项目管理员可查看本项目风险，侧栏无“系统管理”，直接访问存储集群详情重定向到 `403`；旧 `/capacity-predictions` 重定向到 `/projects`，根菜单没有独立“容量预测”。
