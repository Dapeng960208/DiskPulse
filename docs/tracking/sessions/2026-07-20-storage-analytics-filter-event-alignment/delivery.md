# 存储集群分析筛选与事件口径对齐

## 范围

- 将容量趋势、性能分析和故障分析的时间筛选栏放回各自页签内容区。
- 为关联事件页签提供其支持的状态和事件类型筛选。
- 使故障严重级别统计与系统事件只使用同一组厂商原生日志来源。

## 已完成

- 页面筛选栏不再作为 `ElTabs` 的同级内容渲染，存储分布和资源页签不显示时间筛选。
- 关联事件请求会带上选定的 `status`、`category`。
- `error-severity` 查询排除 `source=diskpulse`，与系统事件、重复故障保持 `netapp`/`isilon` 来源一致。

## 验证

- 聚焦 Vitest：详情页页签筛选、关联事件筛选。
- 聚焦 pytest：故障严重级别来源白名单。

## 未验证范围与风险

- 真实 NetApp/PowerScale 事件采集和生产登录态需在部署环境冒烟确认；本次未改变采集逻辑。
