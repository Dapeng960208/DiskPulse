# Dashboard 图表化改版设计

## 目标

- 将首页从“选择项目后逐项目组堆叠卡片”改为项目容量驾驶舱。
- 首屏以容量、风险和层级分布图表为主，只保留必要数字与筛选文案。
- 复用现有项目、项目组和项目存储树接口，不新增后端聚合接口或前端依赖。

## 范围

- 保留 `/` 路由与项目筛选边界。
- 自动选择首个可用项目；用户仍可切换或清空项目。
- 并行读取项目详情、项目组列表和项目存储树。
- 展示总容量环图、项目组容量横向堆叠图、使用率风险分布环图和存储层级矩形树图。
- 不新增全站跨项目汇总、告警时间趋势或实时自动刷新。

## 页面结构

1. 紧凑页头：标题“容量概览”、更新时间、项目选择器。
2. 数字轨道：总容量、已用、可用、项目组数量。
3. 主画布：左侧总容量环图，右侧项目组容量对比。
4. 次画布：左侧风险分布，右侧存储层级分布。

桌面端采用不对称双列布局，移动端回落单列。图表容器是实际信息单元，不做卡片嵌套；颜色继续使用现有 CSS token，蓝色表达容量，橙红色只表达 80% 以上风险。

## 数据流

- 页面挂载时调用 `projectApi.fetch({ page: 1, size: 1, status: 1 })` 获取默认项目。
- 项目变化后并行调用：
  - `projectApi.fetchById(projectId)`；
  - `groupApi.fetch({ project_id: projectId, page: 1, size: 100 })`；
  - `projectApi.fetchStorageTreeById(projectId, { value_type: 'used' })`。
- 项目与项目组容量统一按现有 GB 字段计算，页面格式化为 TB/GB。
- 风险区间固定为 `<80%`、`80–90%`、`90–95%`、`≥95%`，与现有告警阈值一致。
- 项目组容量图按使用率降序取前 10 个，避免标签拥挤；矩形树图保留完整层级数据。

## 状态与错误

- 请求期间显示骨架和图表加载态。
- 任一项目请求失败时清空本次结果并显示一次“加载容量概览失败，请稍后重试”。
- 没有项目或项目没有容量数据时显示简短空态，不渲染零值装饰图表。
- 通过请求序号忽略快速切换项目产生的过期响应。

## 测试与验收

- RED：新增页面测试，锁定默认项目、三路并行请求、数字摘要、四类图表数据、失败和空态。
- GREEN：聚焦 Vitest、目标 ESLint、生产构建通过。
- 响应式检查覆盖 375px、768px、1024px、1440px，无页面级横向滚动。
- 未连接真实后端时，只确认接口调用契约和前端渲染；真实容量值与大数据量布局留作部署验收。

## 取舍

- 不新增 Dashboard 专用后端接口：现有接口已经覆盖单项目驾驶舱，避免重复聚合口径。
- 不引入新图表库：复用已安装的 ECharts。
- 不做实时轮询：当前数据更新时间由采集链路决定，用户切换项目或重新进入页面即可刷新。
