# 四维容量耗尽风险前端

## 信息架构

容量预测不出现在根导航。`/capacity-predictions` 保留为隐藏兼容路由并重定向到项目列表；旧的项目组和用户目录预测深链分别重定向到对应资源详情。

“耗尽风险”放在以下详情页：

- 存储集群详情：仅超级管理员可进入的管理路由，直接展示页签；
- 项目详情：全局发布后向有项目读取权限的用户展示；
- 项目组详情：全局发布后向有项目读取权限的用户展示；
- 用户目录详情：全局发布后向有项目读取权限的用户展示。

项目、项目组和用户目录先读取发布可见性；检查完成且结果为可见时才创建页签。风险面板在用户打开页签后懒加载，不预取完整预测。

## 展示契约

共享 `CapacityExhaustionRiskPanel` 只读取[轻量风险接口](./backend.md#轻量风险接口)，固定展示三项：

1. 耗尽风险；
2. 预计耗尽；
3. 判断依据。

不得在详情中增加曲线、MAPE、模型版本、预测来源、容量计划或关联事件。数据不足、30 日内无风险、无结果、无权限和请求失败必须是不同状态；前端只映射标签样式和日期格式，不重新计算风险等级。

## 管理提示

“系统管理 → 预测治理”必须同时展示“内置基线标准”“风险分级标准”“AI 候选与启用标准”。新增候选的下拉框只列出 AI 中心中已启用的模型；候选表优先显示模型名称并保留 ID 回退文本。

“Theil-Sen 趋势与残差分位”术语旁提供可通过鼠标悬停和键盘聚焦触发的 Tooltip。提示以非算法用户可理解的方式说明：两两日期斜率的中位数用于降低单日异常值影响，历史实际值与趋势线的残差分位用于形成偏低、典型和偏高三种未来范围。完整计算步骤和适用限制以[专题概览](./overview.md#theil-sen-与残差分位如何工作)为准。

Mock 运行时提供四个维度的轻量风险响应，并保持存储集群仅超级管理员、其他维度按项目隔离的权限语义。
