# 前端 Mock RBAC 设计

## 目标

通过 `VITE_USE_MOCKS=true` 提供仅浏览器内存的演示数据和四角色权限体验，默认真实 API 保持不变。

## 架构

- Axios adapter 拦截常规请求；AI 流式请求走同一 Mock 会话的事件序列。
- 种子数据覆盖项目、项目组、用户目录、存储资源、告警、事件、审计、AI 会话和 Dashboard 图表。
- `superadmin` 访问全局和系统管理；项目角色按所属项目和 `capabilities` 限制读取、编辑、成员和限额行为。

## 安全边界

- 演示账户、密码与数据均为虚构且仅在显式 Mock 环境变量下生效。
- 不访问 LDAP、设备、通知、生产 API 或持久化存储；刷新页面重置数据。
