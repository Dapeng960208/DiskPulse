# 项目级 RBAC 与统一审计前端

## 项目详情信息架构规范

- 项目详情必须保留“项目使用实时”并新增“存储分布”；项目列表直接展示项目表格，不保留冗余页签。存储分布页签不提供删除操作。
- 项目使用实时固定在当前项目上下文；项目使用实时和存储分布中的趋势图、告警表、树形图必须填满页签剩余内容区，不使用固定的可视区高度。
- 存储分布不得提供前端筛选条件，固定读取 `GET /projects/{project_id}/storage-tree?value_type=used` 的当前数据库记录，并按“项目组 → 用户”展示。
- 项目组、用户目录和成员与权限必须提供项目范围内的筛选；存储分布是唯一不带筛选的项目详情数据页签。
- 用户目录必须保留项目组标签、项目组、存储集群、Linux 路径、硬限额、软限额、使用量及硬/软限额使用率；不得展示“项目”“存储类型”列。研发用户名仅在离职用户时展示用户类型标识。
- 项目、项目组和用户目录详情的面包屑必须为“项目 → 项目名 → 资源详情”；用户目录和项目组的末级标签分别为“用户名用户详情”和“项目组名项目组详情”。

## 当前行为

- 根导航只保留“项目”入口；项目组和用户目录从一级菜单隐藏，进入 `/project/:id` 后通过“项目组”和“用户目录”页签按项目查看。原 `/groups`、`/usage` 及详情深链继续有效。
- 项目详情集中提供“项目使用实时、存储分布、项目组、用户目录、成员与权限、项目审计”页签；项目列表直接展示项目表格。成员、项目管理员和审计操作只由后端返回的 capabilities 控制。
- 用户目录所属用户由后端自动补齐为项目 `reader`；管理者仍可添加其他成员或调整其有权管理的角色。
- “存储分布”不提供前端筛选条件，固定读取当前项目 `GET /projects/{project_id}/storage-tree?value_type=used` 返回的数据库当前记录，并以“项目组 → 用户”树形图展示。
- 项目详情的“项目使用实时”和“存储分布”页签均占满可用内容高度；趋势图、告警表和树形图随内容区拉伸。
- 项目使用实时页签通过全高 flex 根节点向实时趋势图和告警表传递可用内容高度；切换页签或内容区滚动后，图表与告警表仍占满筛选和概要信息后的剩余区域。
- 项目组详情、用户目录详情、容量池详情和 Qtree 详情的实时趋势均以主内容区为边界：趋势图和告警表填满其余可用高度，不得延伸到应用页脚区域。
- 项目组、用户目录和成员与权限页签均提供项目范围内的筛选；用户目录展示项目组标签、项目组、存储集群、限额和使用率字段，以及软限额与软限额使用率；不展示项目和存储类型列，研发用户名仅对离职用户展示用户类型标识。
- 项目组、用户目录、成员与权限和项目审计的筛选区与底部分页固定在项目详情内容区内；数据表仅在两者之间纵向滚动。四个页签均以 20 条为默认每页条数，成员与权限在前端对已加载成员分页，其余三个由服务端分页。
- 项目、项目组和用户目录详情在关联项目可用时，面包屑按“项目 → 项目名 → 资源详情”展示；用户目录和项目组的末级标签分别使用“用户名用户详情”和“项目组名项目组详情”。
- 项目组、用户目录和独立容量预测列表采用服务端分页；连续翻页时只有最新请求可更新数据、总数、错误和加载状态。
- 独立“容量预测”列表 `/capacity-predictions` 及用户目录详情中的“容量预测最终结果”由全局发布开关与服务端项目权限共同决定，前端不推断或提升权限。
- 项目组的“调整额度”入口只在后端为超级管理员返回 `capabilities.adjust_quota=true` 时展示。用户目录的“调整额度”入口位于项目详情“用户目录”页签表格的最右侧操作列，并要求后端为超级管理员或该项目负责人返回 `capabilities.adjust_quota=true`；详情页不重复显示该入口。“配额历史”同样要求该能力。
- 用户目录详情的“关联事件”只遵守资源项目权限，不复用容量预测发布开关。
- 系统管理中的统一审计入口为 `/admin/audit-events` 及详情路由，只对超级管理员展示。
- 统一审计列表和详情将主体、资源和项目显示为可读关联：主体优先使用当前用户显示名；可访问的项目组、用户目录、存储集群、存储空间和 Qtree 使用轻量资源链接。项目列命名为“关联项目”，明确区分审计记录直接绑定的项目与经资源反查的项目，并显示资源到项目的关联路径；没有当前项目关联时显示“无项目关联”，不以 `-` 掩盖语义。
- AI 新建会话不提交 `project_id`；页面展示的数据仍由服务端当前项目授权过滤。

前端隐藏、禁用和路由守卫只改善体验，不能替代服务端项目隔离。后端模型和 API 边界见 [RBAC 与审计后端说明](./backend.md)。
