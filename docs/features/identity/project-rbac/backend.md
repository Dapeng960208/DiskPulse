# 项目级 RBAC、统一审计与 AI 数据隔离

## 角色与能力

`project_memberships` 为项目成员保存 `reader`、`editor` 和 `project_admin`。超级管理员拥有全局访问权；项目管理员可以管理本项目的 `reader` 和 `editor`，只有超级管理员可以授予或撤销项目管理员。项目组负责人仅可调整其负责项目组及其用户目录的配额。

前端只消费后端返回的 `capabilities`，后端仍在路由和服务层执行最终授权。

## 资源隔离

项目、项目组、用户目录、大文件和可反查告警在数据库分页、统计和导出前按项目过滤。没有项目作用域的存储集群、容量池、存储空间和 Qtree（NetApp）仅向超级管理员开放。

成员 API 位于 `/storage-pulse/api/projects/{project_id}/members`，支持集合 `GET/POST` 和单成员 `PATCH/DELETE`。非超级管理员访问不存在或无权的项目时使用统一拒绝语义，避免枚举项目是否存在。

## 统一审计与 AI

`audit_events` 为追加式审计记录，关联 `X-Request-ID`、`X-Trace-ID` 和操作 ID。认证、成员变更、配额、AI、模型治理、采集、通知及管理写入保留可关联的脱敏摘要；审计记录不暴露凭据、令牌、原始 prompt/response 或完整敏感路径。

AI 会话属于创建者，不绑定项目。工具调用使用当前用户的原路由授权；读取历史或继续对话时重新计算项目可见范围。权限已撤销、范围不可证明或旧记录缺少可见性信息时，助手内容和工具结果默认隐藏。

统一审计 API 为 `GET /storage-pulse/api/v1/audit-events` 及单条详情。超级管理员可查询全局，项目管理员只能查询其授权项目范围。

真实数据库授权、备份恢复和外部设备联调仍需在部署环境验收。
