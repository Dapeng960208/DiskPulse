# 项目级 RBAC、统一审计与 AI 数据隔离

## 角色与能力

`project_memberships` 为项目成员保存 `reader`、`editor` 和 `project_admin`。超级管理员拥有全局访问权；项目管理员可以管理本项目的 `reader` 和 `editor`，只有超级管理员可以授予或撤销项目管理员。项目组的配额调整仅限超级管理员；项目负责人可调整其负责项目内用户目录的配额，但不能调整项目组配额。

创建或更新项目负责人时，服务端在同一事务内将新负责人设为 `project_admin`，并移除旧负责人的 `project_admin` 角色。旧负责人仍拥有该项目用户目录时降为 `reader`，否则移除其成员关系；因此角色联动不会中断目录所有者必须保留的项目访问。超级管理员可独立添加项目管理员；项目管理员可查看项目内全部受项目范围保护的内容，但没有项目组额度调整权限。

用户目录所属用户自动拥有该目录当前项目的 `reader` 成员关系。新增或编辑用户目录、采集同步以及项目组改绑项目时都会幂等补齐；已有 `editor` 或 `project_admin` 不降级。目录仍存在时成员 API 拒绝移除该用户的项目关系。迁移 `000000000014` 为历史用户目录补齐缺失关系；数据迁移不自动撤销旧项目成员关系，避免用户仍有其他目录时误删合法访问。

前端只消费后端返回的 `capabilities`，后端仍在路由和服务层执行最终授权。

## 资源隔离

项目、项目组、用户目录、大文件和可反查告警在数据库分页、统计和导出前按项目过滤。没有项目作用域的存储集群、容量池、存储空间和 Qtree（NetApp）仅向超级管理员开放。

容量耗尽风险按当前资源归属授权：存储集群仅超级管理员可读；项目、项目组和用户目录要求当前项目 `reader` 及以上，并受 `capacity_prediction_settings.user_visible` 控制。预测快照中的历史 `project_id` 不能替代当前资源归属事实。资源关联事件只校验当前项目 `reader` 权限，不复用预测发布开关。完整口径见[四维容量耗尽风险](../../ai/capacity-prediction/overview.md)。

成员 API 位于 `/storage-pulse/api/projects/{project_id}/members`，支持集合 `GET/POST` 和单成员 `PATCH/DELETE`。非超级管理员访问不存在或无权的项目时使用统一拒绝语义，避免枚举项目是否存在。

## 统一审计与 AI

`audit_events` 为追加式审计记录，关联 `X-Request-ID`、`X-Trace-ID` 和操作 ID。认证、成员变更、配额、AI、模型治理、采集、通知及管理写入保留可关联的脱敏摘要；审计记录不暴露凭据、令牌、原始 prompt/response 或完整敏感路径。

AI 会话属于创建者，不绑定项目。工具调用使用当前用户的原路由授权；读取历史或继续对话时重新计算项目可见范围。权限已撤销、范围不可证明或旧记录缺少可见性信息时，助手内容和工具结果默认隐藏。

统一审计 API 为 `GET /storage-pulse/api/v1/audit-events` 及单条详情。超级管理员可查询全局，项目管理员只能查询其授权项目范围。两条 `GET` 同时以 `list_audit_events`、`get_audit_event` 注册给 AI 作为常规只读工具；内部 ASGI 执行复用当前用户 Token 和本段既有授权，不因 AI 工具注册扩大范围。公有响应中的 `resource_id` 固定为 `string | null`；数据库保存的整数逻辑 ID 仅在序列化输出时转换为字符串。

审计查询在完成当前页分页后，批量补齐不写回数据库的关联摘要：`actor`（当前显示名）、`project`（直接绑定项目）、`resource`（当前资源名称）、`related_projects`（经资源反查项目）与 `relation_path`。项目组、用户目录、存储集群、存储空间、Qtree 和存储告警可反查到当前项目；资源已删除或没有可证明关联时保留原始 `resource_type/resource_id`，关联摘要为空。关联项目只按当前仍存在的业务实体计算，不能改变追加式审计历史；项目管理员的间接项目列表会再次限制在已授权项目内，不能借共享存储资源枚举其他项目。

真实数据库授权、备份恢复和外部设备联调仍需在部署环境验收。
