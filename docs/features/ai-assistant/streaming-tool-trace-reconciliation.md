# AI 助手流式回复与工具轨迹持久化对账（实施中）

> 状态：实现进行中。本专题定义本轮已批准的接口与行为目标；后端/前端聚焦测试和登录态浏览器验收完成前，不应将其描述为已发布能力。

## 1. 目标与范围

AI 助手的一次提问必须绑定到同一条助手回复：文本继续通过 SSE `delta` 实时逐字输出，工具调用在该回复下实时显示；请求结束后以服务端持久化记录原位对账。刷新、重新打开会话后，工具轨迹仍准确归属原助手回复。

- 本轮不新增业务表，复用既有 AI 消息和审计持久化。
- 本轮不扩大工具范围或权限；工具仍只调用已显式开放的只读 JSON 接口，并继承当前登录用户的 Bearer Token 与数据范围。
- 输入框固定、消息区高度和滚动等聊天布局优化属于后续独立工作，不在本专题范围内。

## 2. 稳定关联标识与 SSE 契约

服务端在开始输出 SSE 前创建空的助手回复和运行中审计，并返回稳定标识：

- `turn_id`：一次用户提问及其助手回复的稳定标识；本轮全部 SSE 事件均携带它。
- `message_id`：预先创建的助手消息标识；前端从流开始到历史恢复均使用同一消息对象。
- `call_id`：一次具体工具调用的稳定标识。同名工具在同一轮或不同轮被多次调用时，前端只能按 `call_id` 更新，不能按工具名猜测最近一项。

`accepted` 返回 `turn_id`、`message_id`、会话和审计标识。`delta`、`status`、`tool_call_started`、`tool_call_finished`、`completed`、`error`、`cancelled` 都携带 `turn_id`；两个工具事件还必须携带 `call_id`。`completed`、`error`、`cancelled` 返回同一助手消息的最终状态和可恢复工具轨迹。

## 3. 实时显示与原位对账

1. 前端收到 `accepted` 后立即建立目标助手消息；工具尚未输出文字时显示“正在查询数据”。
2. 每个 `delta` 只追加到该消息内容，保持逐字输出效果。
3. `tool_call_started` 在该回复下插入“调用中”条目，展示调用顺序、工具名和参数摘要；`tool_call_finished` 以相同 `call_id` 原位更新成功或失败、耗时和结果摘要。
4. 最终事件只将持久化字段合并到该消息，不替换整条 DOM 或重新创建消息组件，保持已展开的工具详情和用户当前滚动位置。
5. 用户在消息区底部时跟随流式更新；用户主动向上滚动后暂停自动跟随。该交互约束仅描述回复流本身，不包含后续聊天布局改造。

## 4. 持久化、历史恢复与终态

- 审计详情在工具开始和结束时同步保存调用顺序、轮次、`call_id`、工具名、参数、状态、耗时、结果摘要及 `truncated` 标记。
- 工具结果按可展示 JSON 的 UTF-8 序列化大小限制为每次最多 `32 KiB`；超过上限时明确设置 `truncated=true`，客户端不得把该结果当作完整数据。
- 服务端累计已输出的全部 `delta`；正常结束时将相同文本写入预创建助手消息并标记为 `succeeded`。
- 客户端取消时保留已吐出的文本和已完成工具轨迹，助手消息和审计标记为 `cancelled`。连接关闭后浏览器可能收不到终态事件，页面应本地标记“已停止”，服务端仍在关闭处理里落库。
- Provider、工具或生成异常时保留已吐出的文本和已完成轨迹，标记为 `failed`；若尚无文本，显示友好的失败说明。
- 会话详情的每条助手消息返回 `turn_id`、`status` 和 `tool_calls`。旧消息没有关联审计时作为普通成功回复返回，保持历史兼容。

助手消息中的目标形状如下；`tool_calls` 仅属于该条回复：

```json
{
  "id": 12,
  "turn_id": 34,
  "role": "assistant",
  "status": "succeeded",
  "content": "持续流式生成的回答",
  "tool_calls": [
    {
      "call_id": "call_1",
      "tool_name": "list_projects",
      "status": "succeeded",
      "arguments": {},
      "elapsed_ms": 125,
      "result": {},
      "truncated": false
    }
  ]
}
```

## 5. 权限与隐私边界

- 工具调用继续以发起对话的用户身份执行；持久化工具结果不会成为绕过原业务接口权限的读取通道。
- 会话详情仅向会话所有者返回工具参数和可展示结果；跨用户会话查询继续返回 `404`。
- 超级管理员审计列表与详情保持脱敏摘要，不因本专题额外暴露会话用户的工具参数或完整结果。
- 审计可展示的工具轨迹不得记录 API Key、Bearer Token 或未脱敏凭据；工具输出若含敏感字段，仍须在写入可展示审计详情前遵循既有脱敏约束。用户消息的既有持久化边界不因本专题改变。

## 6. 验证与未验证范围

- 后端聚焦：预创建助手消息、`turn_id`/`call_id` 一致性、同名多次调用、`32 KiB` 截断、成功/失败/取消落库、会话所有权和旧消息兼容。
- 前端聚焦：实时 `delta` 与工具状态并行更新、终态原位对账、刷新后的折叠轨迹、停止生成和失败显示、主动上滚后的自动滚动暂停。
- 完成后运行 AI 后端与前端聚焦测试、相关静态检查、生产构建和 `git diff --check`。
- 当前未验证范围：真实登录态浏览器中的长回答、断流、刷新与工具详情展示；该验收须在本轮实现与自动化聚焦测试通过后执行。
