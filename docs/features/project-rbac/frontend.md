# 项目级 RBAC 与统一审计前端

## 当前页面

- 项目详情增加“成员”和“项目审计”页签，用于查看项目成员、角色及项目范围审计记录；写入按钮由前端角色状态控制，后端仍负责最终授权。
- 系统管理增加“统一操作审计”列表和详情路由：`/admin/audit-events`、`/admin/audit-events/:id`。路由守卫仅向 `superadmin` 展示入口。
- AI 新建会话请求支持可选 `project_id`，页面可将会话与项目关联；未提供项目时保持历史兼容行为。

## 使用边界

- 前端隐藏、禁用或路由守卫不是安全机制；项目成员、项目审计和统一审计的可见范围必须以服务端权限过滤为准。
- 统一审计页面只能展示已经被后端实际写入的 `audit_events`。当前业务写操作尚未全面接入审计，因此列表不应被解释为完整操作历史。
- 项目成员 HTTP Router 已注册到主应用；真实增删改仍需在登录态、角色矩阵和项目隔离条件下完成端到端联调。
