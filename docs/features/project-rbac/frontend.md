# 项目级 RBAC 与统一审计前端

## 当前页面

- 根导航只保留“项目”入口；项目组和用户目录从一级菜单隐藏，进入 `/project/:id` 后分别通过“项目组”和“用户目录”页签按项目范围查看。原 `/groups`、`/usage` 及详情深链仍保留，便于已有链接继续使用。
- 项目详情同时提供“容量概览”“成员与权限”和“项目审计”页签；成员、项目管理员和审计操作只由后端返回的 capabilities 控制，后端仍负责最终授权。
- 系统管理增加“统一操作审计”列表和详情路由：`/admin/audit-events`、`/admin/audit-events/:id`，对应后端 `/storage-pulse/api/v1/audit-events`。路由守卫仅向 `superadmin` 展示入口。
- AI 新建会话不要求也不保存 `project_id`；会话归创建者所有，AI 工具返回的数据由当前用户的后端项目权限过滤。

## 使用边界

- 前端隐藏、禁用或路由守卫不是安全机制；项目成员、项目审计和统一审计的可见范围必须以服务端权限过滤为准。
- 统一审计页面只能展示已经被后端实际写入的 `audit_events`。当前纳入范围的认证、成员、配额、AI、模型治理、采集、通知和管理写入均已接入；生产部署前的外部设备/数据库联调仍待完成。
- 项目成员 HTTP Router 已注册到主应用；真实增删改仍需在登录态、角色矩阵和项目隔离条件下完成端到端联调。
- 项目详情的成员/项目审计页签和项目组、用户目录的配额按钮只读取后端返回的 `capabilities`；前端不推断或提升权限。
- 独立“容量预测”列表及用户目录详情中的“容量预测最终结果”页签由全局发布开关与服务端项目权限共同决定；用户详情首次切换时才加载预测数据。列表、资源预测和容量计划接口均在后端反查当前项目权限，关闭全局发布时非超级管理员固定收到拒绝响应。用户详情的“关联事件”只遵守资源项目权限，不复用预测发布开关。
- 项目详情中的项目组、用户目录和独立容量预测列表都采用服务端分页；前端用请求序号保证只有最新请求可更新当前页数据、总数、错误和加载状态。
