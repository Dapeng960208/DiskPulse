# 项目级 RBAC、统一操作审计与 AI 数据隔离实施记录

## 数据库与模型

- 合并原成员和审计迁移为显式前向 `000000000009_project_rbac_unified_audit.py`；它以既有遥测账本 r8 为上游，避免重写已可能执行的迁移。
- 新增 `project_memberships` 和 `audit_events`；删除 `Project.pt_user_id`。
- SQLite batch 操作通过 `copy_from` 保留 `projects.storage_alert_rule`；三方言离线 DDL 和 SQLite 实际升级/降级均已覆盖。

## 授权与资源接入

- `project_access_service` 集中成员角色、项目反查、资源权限和 UI capabilities。
- 项目、项目组、用户目录、告警、大文件、Dashboard、导出和趋势读取在分页/统计前执行项目限制；无作用域资源限制为超级管理员。
- 采集服务按用户目录归属幂等创建 `reader`；配额服务拒绝无认证调用，并保留项目组负责人例外。

## 审计与关联

- 关联中间件在普通和异常响应上回写请求/追踪 ID。
- 统一审计接入认证、成员、配额、AI 会话与消息、AI 模型治理、采集、通知和管理写入 fallback。
- 审计服务、AI 轨迹和 AI 管理审计详情均执行脱敏；触发器阻止审计更新和删除。

## AI 隔离

- AI 会话未增加项目字段。工具执行保存项目范围元数据，而非为会话绑定项目。
- 历史读取重新计算当前成员权限；失效或不可证明范围的项目工具回合返回受限占位，不返回助手文字或工具结果。模型续聊输入采用同一过滤；无工具总结会继承既有会话范围，缺少 audit/visibility 的旧助手回合 fail-closed。
- `AIAuditLog.trace_id` 优先使用请求上下文，工具轨迹与管理员详情均去除敏感字段。

## 合并前安全复查修正

- 审计通用脱敏名单增加 `response`、`raw_response`、`device_response`、`body` 等响应别名，防止原始模型或设备响应仅因字段名称不同而持久化。
- 撤销成员资格后，AI 历史不能再作为模型续聊上下文泄露；服务会以安全占位替代无权助手正文，并将当前回合 visibility 与历史审计范围合并。

## 前端

- 新增项目成员和项目审计页签、统一审计列表/详情路由与审计 API client。
- 用户目录、项目组列表只依据后端 capabilities 呈现配额动作；AI 新建会话不传递 `project_id`。

## 提交与测试方式

每项审查发现使用 RED 测试提交后再最小 GREEN 提交。迁移、日志关联、AI 轨迹、AI 模型审计、配额认证和前端能力显示均保留独立检查点，便于追溯。
