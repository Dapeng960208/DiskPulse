# 应用页脚固定布局设计

## 状态

- 2026-07-17 已按批准设计实现；自动化测试、覆盖率、目标 ESLint 和生产构建通过。
- 按用户后续要求跳过完整浏览器滚动验收。

## 目标

- 将版权页脚固定为应用壳右侧工作区的独立底部区域，高度统一为 `40px`。
- 侧栏显示时，页脚左边缘贴合侧栏右边缘；无侧栏时，页脚铺满工作区宽度。
- 主内容独立滚动，滚动行为不改变页脚在视口中的位置，也不允许页脚覆盖业务内容。

## 布局设计

- 应用头部下方继续使用“侧栏 + 右侧工作区”的横向布局。
- 右侧工作区改为“主内容 + 页脚”的纵向布局；`AppFooter` 从主内容 `ElScrollbar` 中移出，放在 `ElMain` 后面。
- 主内容、工作区和滚动容器使用弹性高度及 `min-height: 0`，由主内容占用扣除页脚后的剩余高度。
- 页脚保持普通文档流布局，不使用 `position: fixed`，避免重复维护侧栏展开、折叠和无侧栏状态的水平偏移。

## 实现边界

- 只调整 `AppLayout`、`AppFooter` 及对应前端合同测试。
- 保留版权文字、动态年份、居中方式、主题颜色和上边框。
- 不修改路由、业务页面、后端接口、数据结构或既有窄屏侧栏策略。

## 验证

- TDD 锁定页脚高度、在工作区中的层级位置、与内容滚动容器的分离，以及无侧栏状态。
- 执行聚焦 Vitest、目标 ESLint、全量覆盖率、生产构建和 `git diff --check`。
- 完整浏览器验收原计划覆盖 `/usage` 的页脚几何、内容滚动和侧栏展开/折叠；实施阶段按用户后续要求停止，未将该场景列为已验证。
