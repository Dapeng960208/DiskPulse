# 详情页动态面包屑

## 目标

所有详情页展示完整资源层级，并在最后一项显示当前对象名称和“详情”。例如用户目录显示“用户目录 / `rd_username`详情”，项目组显示“项目组 / 项目组名称详情”。

## 规则

- 路由元数据声明稳定的父级层级；页面加载对象后仅替换最后一项详情文本，不改变路由、权限或导航入口。
- 用户目录优先使用 `user.rd_username`；响应缺失时回退 Linux 路径最后一级。
- 项目、项目组、容量池、存储空间、Qtree（NetApp）和存储集群使用当前详情响应的 `name`；AI 审计使用稳定的“审计记录 #ID”。
- 面包屑文字始终保留完整 HTML `title` 提示；视口不足时仅截断显示文本，层级与完整名称不丢失。
- 加载失败或对象名称为空时保留路由的静态“详情”标题，不展示错误或旧页面遗留的名称。

## 验证

- 单元测试覆盖静态层级、动态对象名替换、文本提示数据和路由隔离状态。
- 详情页聚焦测试确认实时详情、项目详情和存储集群详情可在既有请求完成后写入对象名。
- 浏览器验收检查长名称截断时的 `title` 属性与显示层级。
