# 存储告警规则

## 范围与角色

存储告警对用户目录、项目组和项目的最新采集值执行规则评估，并以持久化事件驱动异步飞书投递。超级管理员配置系统、项目和项目组规则并查看历史；业务收件人分别为用户、项目组负责人和项目负责人。

## 规则解析

- 系统规则必须完整；项目和项目组要么继承，要么覆盖整份规则，不进行字段级合并。
- 用户目录按“项目组 -> 项目 -> 系统”解析，项目组按“项目组 -> 项目 -> 系统”解析，项目按“项目 -> 系统”解析。
- 规则包含 `hard` 或 `soft` 口径，以及严格递增的重要、严重、紧急阈值和正整数重复间隔。
- 软口径没有有效软限额时完全跳过本轮，不回退硬限额，也不改变已保存状态。

## 评估与状态

- 只评估本轮真正刷新成功的对象；失败集群或过期数据不触发、恢复或累计连续次数。
- 首次超限仅累计，第二次连续超限产生触发事件；升级立即通知，降级静默更新，同级到期后重复通知，恢复正常后通知一次并清空活动状态。
- 有效规则改变时静默重置状态，不制造恢复事件。
- Redis 锁与数据库唯一约束共同避免并发重复评估；告警状态与事件在同一事务内持久化，提交后才投递通知。

## 投递与安全

投递在采集和状态事务之外执行。全局抄送覆盖全部事件，项目组抄送仅覆盖其范围；紧急事件额外通知存储管理员。收件人去空、保序去重，无收件人或功能关闭时标记为 `skipped`。失败使用有限重试，最终状态为 `failed`，不回显密钥、令牌或设备响应。

飞书通知按“资源信息、容量概览”分段展示，不重复显示事件类型和告警采用口径。硬限额与软限额使用统一容量显示单位；各自利用率按当前有效规则的阈值范围显示颜色标识：低于重要阈值为绿色，达到重要阈值为黄色，达到严重阈值为橙色，达到紧急阈值为红色。

## API 与页面

- 超级管理员通过存储配置、项目和项目组入口维护规则；前端只展示服务端验证后的继承预览。
- 告警列表支持事件类型、配额口径和投递状态筛选，保持数据库分页。
- 用户目录详情 `/usage/:id` 的容量趋势首屏已展示该目录的关联告警；不再提供重复的“告警”页签或额外告警查询。
- 公开响应不返回收件人快照、投递错误详情或通知凭据。

真实 Redis、Celery、飞书、多数据库升级和登录态页面仍需在部署环境验证。
