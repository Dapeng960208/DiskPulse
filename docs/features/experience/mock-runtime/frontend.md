# 前端 Mock 演示

在 `frontend/` 目录运行 `pnpm mock` 启用内存 Mock 网关。默认运行模式继续请求真实后端，Mock 只能通过显式模式启用。

Mock 覆盖超级管理员、项目管理员、编辑者和只读用户的代表性列表、详情、图表和写操作状态。响应保持前端 API 所需的 `data`、`meta`、`traceId` 结构，并且只包含虚构数据。

- 登录页的演示账户快捷入口仅在 `VITE_USE_MOCKS=true` 的 Mock 模式渲染；真实 API 模式不展示，也不预填演示凭据。
- 系统资源 CRUD 仅对超级管理员开放；项目资源继续遵循后端兼容的 capability 模型。
- Mock 适用于界面演示、状态覆盖和前端回归，不绕过真实路由、服务端授权或项目隔离。
- 新增页面或可见字段时，同步补充代表性 Mock 数据，避免以空壳数据掩盖契约缺失。
