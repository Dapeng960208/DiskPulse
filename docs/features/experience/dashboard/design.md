# Dashboard 图表化改版设计

## 目标

- 首页提供“全局总览 + 项目下钻”，默认展示近 30 天数据。
- 页面减少说明文字，以容量趋势、容量对比、使用率和告警级别饼图为主要信息载体。
- 只使用现有存储集群、项目、项目组和告警数据，不引入业务数据分类、数据库迁移或新前端依赖。

## 页面结构

1. 页头：标题“存储概览”、可清空项目选择器和固定“近 30 天”标识。
2. 摘要轨道：容量上限、已使用、可使用、近 30 天告警数。
3. 主画布：容量趋势折线图和容量使用率环图。
4. 次画布：容量横向堆叠对比和近 30 天告警级别饼图；项目视图同时展示用户容量排行。

全局视图显示“物理总容量”和“项目容量对比”；选中项目后，同一画布切换为“项目限额”和“项目组容量对比”。桌面端使用不对称双列布局，`1024px` 以下图表回落单列，`768px` 和 `375px` 继续压缩筛选器与摘要轨道。颜色、边框、间距和明暗主题全部复用 `variables.scss` token。

## 接口与口径

Dashboard 不使用覆盖整页的聚合接口，每个数据块使用独立认证接口：

- `GET /storage-pulse/api/dashboard/summary`：摘要和当前范围，可选正整数 `project_id`。
- `GET /storage-pulse/api/dashboard/capacity-trend`：每日已使用容量，可选正整数 `project_id`。
- `GET /storage-pulse/api/dashboard/capacity-items`：按已使用容量降序的前 10 个项目或项目组，可选正整数 `project_id`。
- `GET /storage-pulse/api/dashboard/alert-levels`：近 30 天按 `alert_level` 分类的告警数量，可选正整数 `project_id`。
- `GET /storage-pulse/api/dashboard/top-users`：项目内用户已用容量 Top 10，必须传正整数 `project_id`。

数据口径：

- 全局摘要使用启用存储集群的物理容量快照；全局容量对比使用非公共项目配额，两者不要求相加相等。
- 项目摘要使用项目配额；项目容量对比只包含启用监控的项目组。
- 项目用户排行通过用户目录关联项目组和项目，按用户汇总已用容量；用户名优先显示研发用户名。
- 告警只统计 `source=diskpulse`、`alert_type=alert`、`event_type=trigger`，按现有“重要、严重、紧急”级别分类，排除配额调整和厂商事件。
- 项目告警范围包含项目、所属项目组和所属用户目录。
- QuestDB 不可用时容量趋势为空，PostgreSQL 容量快照、对比项和告警数据仍正常返回。
- `project_id <= 0` 返回 `422`，项目不存在返回 `404`。

## 前端组件

- `DashboardChart`：懒加载 ECharts，接收标准 option，负责初始化、响应式 resize、更新和卸载释放。
- `PieCharts`：保留原夜莺图默认契约，新增 `dashboard` 环图模式与中心使用率标签。
- `ProjectSelect`：新增可选 placeholder，默认文案不变；Dashboard 使用“全部项目”。
- 页面用请求序号忽略快速切换产生的过期响应；各摘要和图表独立加载、独立显示空态，部分接口失败不阻塞其他数据块。
- 根画布固定从顶部开始排列，项目切换期间不允许 Grid 行拉伸形成大面积空白。
- 项目视图第二排按 `2:2:1` 展示“项目组容量对比 + 用户使用 Top 10 + 告警级别”；全局视图继续按双列展示“项目容量对比 + 告警级别”，`1024px` 以下统一回落单列。

## 验证边界

Dashboard 的接口口径、局部失败降级、项目范围和图表生命周期需要自动化覆盖。真实 QuestDB 查询、登录态浏览器和生产规模布局仍需在部署环境复验。

## 取舍

- 新增单一聚合接口，避免浏览器并发拼接多种实体后产生口径漂移。
- 不增加告警级别分类、容量预测、成本分析、厂商性能指标或自动轮询。
- 不引入新图表库，继续使用项目已安装的 ECharts。
