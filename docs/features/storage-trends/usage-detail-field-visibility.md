# 用户目录详情扩展字段隐藏设计

## 目标

用户目录详情的“容量趋势”页签暂时只展示第一行的目录、限额、使用量和利用率。当前第 2–4 行的文件、权限、时间和 inode 等扩展字段不展示，但保留代码，便于数据补齐后恢复；详情同时提供关联数据页签，避免把预测和事件混入实时趋势。

## 实现边界

- 在 `UsageDetailPage.vue` 中使用 Vue 模板注释包住 `extra-descriptions` 插槽内的 12 个扩展字段 `ElDescriptionsItem`，不删除字段表达式。
- `UsageDetailPage.vue` 以懒加载页签展示“容量趋势、配额历史、容量预测最终结果、关联事件、告警”；切换页签后才读取对应接口，预测和事件保持资源级隔离。
- `ElDescriptionsItem` 仍用于“容量预测最终结果”摘要，因此只注释扩展字段模板，不再注释该组件导入。
- 不修改共享 `RealTimePage`、后端响应、QuestDB/PostgreSQL 数据、筛选器、趋势图或全局告警列表；详情告警页签复用现有告警查询并按 `StorageUsage` 和资源 ID 过滤。
- “配额历史”页签只在后端返回 `capabilities.adjust_quota=true` 时读取，其他账号显示无权限空态；错误和无数据状态不伪装成正常结果。

## 验证

- RED：页面回归测试确认扩展字段目前仍会被渲染。
- GREEN：同一测试确认第 2–4 行字段标签不再出现，同时静态检查确认字段代码仍保留在注释中。
- 原扩展字段隐藏聚焦测试、目标 ESLint 和生产构建已覆盖该隐藏边界；本轮导航与关联数据页签的完整聚焦/浏览器验证仍待交付记录确认。

## 恢复方式

待扩展数据稳定后，取消模板块和导入行的注释即可恢复，字段名和数据绑定无需重写。
