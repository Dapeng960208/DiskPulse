# 用户目录详情扩展字段隐藏设计

## 目标

用户目录实时监控详情暂时只展示第一行的目录、限额、使用量和利用率。当前第 2–4 行的文件、权限、时间和 inode 等扩展字段不展示，但保留代码，便于数据补齐后恢复。

## 实现边界

- 在 `UsageDetailPage.vue` 中使用 Vue 模板注释包住 `extra-descriptions` 插槽内的 12 个 `ElDescriptionsItem`。
- 同步注释当前不再使用的 `ElDescriptionsItem` 导入，不删除字段表达式。
- 不修改共享 `RealTimePage`、后端响应、QuestDB/PostgreSQL 数据、筛选器、趋势图和告警列表。
- 第一行摘要继续使用共享详情信息正常展示。

## 验证

- RED：页面回归测试确认扩展字段目前仍会被渲染。
- GREEN：同一测试确认第 2–4 行字段标签不再出现，同时静态检查确认字段代码仍保留在注释中。
- 执行用户目录详情聚焦 Vitest、目标 ESLint 和生产构建；不使用浏览器自动化。

## 恢复方式

待扩展数据稳定后，取消模板块和导入行的注释即可恢复，字段名和数据绑定无需重写。
