# 全站存储趋势图设计

## 目标与范围

- 用统一趋势组件覆盖容量池、存储空间、Qtree、项目、项目组、用户目录、存储集群容量分析和 Dashboard 容量趋势。
- 单对象趋势同时表达数值变化和当前告警等级，多对象趋势优先保留对象身份辨识。
- 复用现有 ECharts、明暗主题 token 和响应式布局，不引入新图表依赖。

统一组件为 `StorageTrendChart`。实时趋势、Dashboard 和存储集群容量分析均通过相同的 option 构建逻辑生成曲线、阈值、Tooltip、工具箱和空数据画布。旧的 `LineCharts`、`MultipleLineCharts` 不再承担存储趋势展示。

## 视觉规范

### 单对象分段曲线

单对象曲线按有效三级阈值分为四段：

| 区间 | 颜色 | 色值 |
| --- | --- | --- |
| 正常 | 项目蓝 | `#446AEE` |
| 重要 | 金黄 | `#D69B1E` |
| 严重 | 橙色 | `#EF5923` |
| 紧急 | 红色 | `#D3372F` |

相邻点跨越阈值时按时间和值做线性插值，在真实阈值交点插入仅供绘制的点，确保颜色从交点开始切换。曲线不显示面积渐变、最大/最小图钉或常驻数据点。

三级阈值使用对应等级颜色的细虚线，标签为“重要 80%”“严重 90%”“紧急 95%”这类格式，并位于阈值线上方。百分比纵轴固定从 `0%` 到 `100%`，间隔 `10%`；水平网格线使用细虚点线。实际阈值取接口返回值，不固定为示例数字。

### 多对象对比

- 多对象曲线保留 ECharts 项目色板中的对象身份色，不按阈值分段换色。
- 容量使用量对比不绘制阈值线，避免不同对象限额造成错误比较。
- 使用率或告警口径使用率对比使用系统三级阈值，并在 Tooltip 中标注“对比口径：系统阈值”。

### 交互、主题与响应式

- Tooltip 显示时间、对象名称和值；单对象额外显示当前等级、硬/软限额口径和系统/项目/项目组规则来源。
- 工具箱只保留还原和保存图片，不提供数据视图或可编辑标记工具。
- 组件监听根节点主题 class 和容器尺寸；主题变化时重新读取 CSS token，尺寸变化时调用 ECharts `resize`。
- 图表容器提供可访问性名称，卸载时断开观察器并释放 ECharts 实例。
- 百分比阈值标签预留右侧空间；窄容器继续保持画布宽度收缩，不产生页面横向溢出。

## 指标与阈值口径

实时趋势查询支持以下指标：

| `indicator` | 含义 | 阈值行为 |
| --- | --- | --- |
| `used` | 使用容量 | 单对象按当前告警口径限额换算容量阈值；多对象不显示阈值。 |
| `use_ratio` | 硬限额使用率 | 单对象使用有效规则阈值；多对象使用系统阈值。 |
| `alert_ratio` | 告警口径使用率 | 后端按有效规则选择 `used_ratio` 或 `soft_use_ratio` 历史列。 |
| `file_used` | 已用文件数 | 不显示容量阈值；仅用户目录实时趋势支持。 |

有效规则沿用既有优先级：项目组规则 → 项目规则 → 系统规则。存储集群和容量池属于物理容量视图，固定使用硬容量及系统规则；Dashboard 全局趋势同样使用系统规则和硬容量，项目筛选后的 Dashboard 使用项目有效规则。

单对象 `used` 指标使用 `quota_limit_gb × 阈值百分比` 得到三条容量阈值。有效规则选择软限额但对象没有可用软限额时，接口将 `quota_limit_gb` 返回为空，前端不绘制虚构阈值。

## 接口契约

实时趋势、Dashboard summary 和存储集群容量变化响应新增 `trend_meta`，原有数据字段保持兼容：

```json
{
  "trend_meta": {
    "quota_basis": "hard",
    "rule_source": "system",
    "thresholds": {
      "important": 80,
      "serious": 90,
      "emergency": 95
    },
    "quota_limit_gb": 1024.0,
    "ratio_indicator": "used_ratio"
  }
}
```

字段说明：

- `quota_basis`：当前告警口径，取值为 `hard` 或 `soft`。
- `rule_source`：生效规则来源，取值为 `system`、`project` 或 `group`。
- `thresholds`：重要、严重、紧急三级百分比阈值。
- `quota_limit_gb`：当前告警口径的容量限额；不适用或缺失时为 `null`。
- `ratio_indicator`：`alert_ratio` 实际读取的历史列，为 `used_ratio` 或 `soft_use_ratio`。

`indicator` 通过 Pydantic 字面量校验；未支持的值返回 `422`。QuestDB 查询继续使用服务端白名单选择指标列，不接受客户端提供任意列名。

## QuestDB 历史数据边界

本次复用现有 QuestDB 软限额历史能力：迁移 `000000000002_add_soft_quota_metrics.sql`、时序模型和采集任务已经包含可空的 `soft_limit`、`soft_use_ratio`，因此不新增重复迁移。

迁移前的历史样本保持空值。选择 `alert_ratio` 且有效规则采用软限额时，历史缺失显示为数据缺口，不使用当前软限额反推或回填旧数据，也不把缺失值转换成零。
