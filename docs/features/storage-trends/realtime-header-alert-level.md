# 实时监控页头与告警紧急程度展示设计

## 目标

- 删除实时监控页头右侧重复显示的开始、结束时间。
- 告警表不再展示内部提示描述，改为直接展示用户可理解的告警紧急程度。

## 页面行为

- 筛选栏中的时间范围继续作为趋势和告警查询的唯一时间控制；删除页头摘要不改变请求参数。
- 告警表列固定为“告警紧急程度、触发值、时间”。
- `alert_level` 映射为中文：`important` 为“重要”、`serious` 为“严重”、`emergency` 为“紧急”，并兼容历史 `high`、`medium`。
- 紧急、严重和历史高等级使用危险标签；重要和历史中等级使用警告标签；未知值使用信息标签并显示原值或“-”。

## 实现边界

- 只修改共享 `RealTimePage.vue` 的展示和本地枚举映射。
- 不修改时间筛选、API 请求、告警响应、数据库、趋势图或告警排序。
- 容量池、存储空间、Qtree、项目、项目组和用户目录等复用该页面的详情视图同步生效。

## 验证

- TDD 锁定页头不再出现重复时间摘要、提示列消失、紧急程度列及中文映射存在。
- 执行实时监控页聚焦 Vitest、目标 ESLint 和生产构建。
- 按用户此前要求，本轮不执行浏览器自动化验证。
