# 全局列表行操作设计

## 目标

统一全站列表和表格的行操作密度，避免按钮在窄列中换行，同时保持现有权限、禁用条件和业务调用不变。

## 交互与权限

- 行操作最多两个时直接展示；超过两个时保留一个高频操作，其余收入“更多”。“更多”触发按钮计入两个直显操作，菜单项不计入。
- 操作列使用固定宽度、单行和右对齐布局，共享 `.list-row-actions`、`.list-row-actions__more` 与 `.list-row-actions__danger` 全局样式。
- 项目组和用户目录始终直接显示“详情”。
- 通过 `hasRole('disk-monitor:admin')` 的用户额外显示“更多”菜单；现有 `superadmin` 与 `*:root` 继续由统一权限函数兼容。
- 项目组“更多”包含“调整配额”“编辑”“删除”，其中“删除”使用危险态。
- 用户目录“更多”只包含当前已有的“调整配额”，不恢复已禁用的编辑或删除能力。
- “添加项目组”和“新增用户目录”统一只对管理员显示。
- 存储集群直接显示“详情”；管理员可见“更多”中的“编辑”“删除”，新增入口也只对管理员显示。
- AI 模型直接显示“编辑”，“连接测试”“删除”进入“更多”；删除继续显示二次确认。AI 中心继续依赖现有超级管理员路由守卫。

## 实现边界

- 规则只处理列表和表格的行操作，不处理筛选栏、表头、页面工具栏、卡片操作或弹窗底部按钮。
- 复用 Element Plus `ElDropdown`、`ElDropdownMenu`、`ElDropdownItem`，不新增依赖或公共组件。
- 只修改前端入口可见性和共享样式，不修改后端 API、角色类型或授权机制。
- 普通用户不渲染“更多”入口，避免空菜单；管理操作仍由后端权限继续兜底。
