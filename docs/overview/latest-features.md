# 最新功能与修复

## 2026-07-16：存储告警规则与飞书通知

- 系统提供默认规则，项目和项目组可使用完整自定义规则或按“项目组 → 项目 → 系统”继承；规则可选择硬/软限额口径并配置三级阈值和重复频次。
- 告警连续两次超限后触发，支持升级立即通知、降级静默、同级定时重复和恢复通知；只评估当前采集轮成功刷新数据，避免失败集群或重复样本误报。
- 用户目录、项目组和项目分别通知本人、项目组负责人和项目负责人，并支持全局 CC、项目组个人 CC、紧急管理员和 debug 收件人替换。
- 飞书发送通过独立通知微服务异步完成，失败按 1/5/15 分钟重试；告警页可按事件类型、限额口径和发送状态筛选。系统设置、项目和项目组表单均已提供规则配置或继承预览。
- 自动化验证为后端 `312 passed`、前端 `205 passed`，迁移号为 `000000000006`；真实 Redis/Celery、飞书服务、多数据库在线迁移和登录浏览器仍需部署环境验收。
- 完整设计与验收边界见 [存储告警规则设计](../features/storage-alerts/design.md)。

## 2026-07-16：Isilon 按 Directory Quota 路径采集延迟

- 性能采集改读 OneFS `path` performance dataset，通过已固定 workload ID 映射 Directory Quota 完整路径，并按读、写、其他请求计数计算微秒平均值后转为毫秒。
- 性能页固定查询 `object_type=volume`，不再把历史节点磁盘延迟显示成 Isilon 卷延迟。
- 真机 OneFS 9.11.0.5 已验证：当前 `path` dataset 有 8 个已固定 workload，其中 7 个匹配数据库中的 Directory Quota，正式采集写入并回读 7 个路径；另一个父路径由服务端过滤。数据库共有 67 个 Directory Quota，其余 60 个路径需由 root 执行 `isi performance workloads pin path "path:<完整路径>"` 后才可采集。

## 2026-07-16：系统事件支持搜索和分页

- 故障分析中的系统事件支持按时间范围、关键字和日志等级查询，默认每页展示 20 条，可切换为 50 或 100 条并跳转页码。
- 关键字可匹配事件代码、事件对象和内容；查询在数据库分页前生效，总数与当前筛选条件一致。
- “对象”改为“事件对象”：NetApp 优先显示 EMS 节点名称并保留原始节点 UUID，Isilon 将 OneFS `devid` 显示为“节点 N”。

## 2026-07-16：修复 Isilon 性能与故障数据为空

- 性能采集兼容 PowerScale statistics 返回的秒级延迟和 Unix 时间戳，合法的零延迟样本不再被跳过。
- 故障采集兼容 OneFS event group 与嵌套 event list 的真实结构，设备事件可以进入严重级别、重复故障和系统事件分析。

## 2026-07-15：全站筛选栏改为渐进式工具栏

- 全站列表页统一为“主筛选工具栏 + 更多筛选 + 可删除条件标签”；筛选项采用左对齐 label 与等宽控件，每行根据容器宽度自动显示 `1–5` 项，主筛选与高级筛选共享字段轨道并保持上下列宽一致，复杂页面默认只展示最常用条件。
- 项目组和用户目录将次要条件收进“更多筛选”；删除已生效标签会同步清理依赖条件并立即刷新，普通字段仍由搜索显式触发。
- 操作区统一为“更多筛选、重置、绿色导出、蓝色搜索”，导出与搜索相邻，搜索保持最右侧主操作；明暗主题与现有 Element Plus 风格保持一致。

- 存储分布不使用时间范围和健康报告导出，打开该页签时隐藏无关的时间筛选工具栏。

## 2026-07-15：登录 token 改用 Redis 会话缓存

- 登录成功后把 JWT 摘要写入 Redis DB 7，并在每次鉴权时同时校验 JWT 和 Redis 会话白名单。
- JWT 与 Redis 会话默认有效期统一为 7 天；登出会删除当前 token 会话，后端重启不再丢失撤销/会话状态。
- Redis 故障时认证链路返回 `503` 并拒绝放行；Redis 不保存原始 JWT。

## 2026-07-15：存储健康入口收敛与真机采集修复

- 删除一级菜单“存储健康”，统一从“系统管理 → 存储集群”列表最后一列进入详情；页面移除集群选择器和重复的集群配置摘要。
- 时间筛选、搜索/重置和导出统一进入同一工具栏动作区，避免导出按钮与日期范围重叠；空态明确区分“当前时间范围无数据”和“从未成功采集”。
- NetApp 性能采集改用真机 REST API 支持的单数 `metric` 字段，修复请求 `metrics` 返回 `400` 导致 QuestDB 始终无性能数据的问题。
- 普通告警页只展示 DiskPulse 容量、周报和扩容类告警；NetApp/PowerScale 原生事件归到故障分析中的“系统事件”，不再错误显示为“扩容”。
- 当前 PowerScale 配置账号登录 `platform` 服务返回 `403`，事件和性能接口随后返回 `401`；需先在 OneFS 修复账号的 Platform API 登录与只读权限，再重启 Celery worker 验证落库。

## 2026-07-15：修复项目组监控配置无法保存

- 新增编辑回归测试，覆盖后端详情对象直接进入表单后的提交边界。

## 2026-07-15：存储集群健康分析与报表导出

- 错误统计合并可归属当前集群的 DiskPulse 容量告警和 NetApp/Isilon 设备事件；重复故障仅统计设备事件。
- NetApp 按 Volume 分析延迟；PowerScale 仅使用已固定的 path workload 映射 Directory Quota，不再用节点延迟冒充存储空间延迟。

- 功能实现、存储健康聚焦测试和后端/前端全量自动化验证已完成；真实 NetApp、PowerScale、PostgreSQL、MySQL、QuestDB 和浏览器冒烟尚未执行。

## 2026-07-15：AI 助手与超级管理员 AI 中心

- 所有登录用户新增“AI 助手”，支持模型选择、会话恢复、SSE 消息流、停止生成、失败重试、工具状态和安全 Markdown。

## 2026-07-14：登录认证链路去重与 LDAP 轻量化

- 登录页取得用户 profile 后由路由守卫复用，刷新后 store 为空时才重新加载，避免登录跳转重复请求。
- 后端在同一请求内复用已通过 JWT 校验的用户，仍对每个新请求独立校验 JWT，不跨请求缓存认证结果。
- LDAP 连接不再读取无关的完整目录 schema/info，保留 STARTTLS、CA 证书校验和连接超时安全边界。

## 2026-07-14：用户信息管理与 LDAP 一键同步

- 系统管理新增“用户信息管理”入口，仅超级管理员可新增、编辑、删除用户或一键同步 LDAP；现有登录用户列表接口继续供既有选择场景复用。
- 用户类型统一为离职 `0`、公共用户 `1`、在职 `2`；LDAP 同步可新增在职用户、恢复重新出现的离职用户，并把快照中缺失的在职用户标记为离职。
- 公共用户类型只允许超级管理员人工修改；LDAP 仅更新其非空姓名、邮箱和部门，缺失的公共用户不受影响。
- LDAP 空快照、搜索范围不完整或用户名冲突时不写入数据库并回滚；同步不会删除用户，也不增加定时任务、历史表或预演接口。

## 2026-07-14：存储协议与 TLS 校验按集群配置

- 全局 YAML `storage.tls_verify` 已删除；采集客户端和 Isilon Quota 手工检查均读取数据库中的集群配置。
- HTTP 下 TLS 校验不适用，设备凭据会以明文传输，只应在可信隔离网络中使用。

## 2026-07-14：集群配置后自动同步存储资源

- 新建或更新启用的存储集群后，自动投递该集群的采集任务，无需等待下一次全量周期采集。
- NetApp 与 Isilon 共用现有 `StoragePulseMonitor` 链路，只采集本次配置对应的集群。
- 存储集群新增/编辑表单新增“是否启用”开关，新建默认启用，停用集群不触发采集。
- 开发环境补充 Uvicorn 可见的任务投递日志，Celery worker 补充任务开始日志。
- NetApp/Isilon 使用各自集群的 `protocol` 和 `tls_verify`；连接失败不再同步空数据。
- 未启用集群不触发；异步任务失败不回滚已保存配置，并由服务端日志和后续周期任务继续兜底。
