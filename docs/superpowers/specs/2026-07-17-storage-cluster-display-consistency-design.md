# 存储集群与存储类型全站展示一致性设计

## 目标

以用户目录列表为展示基准：存储集群名称使用普通文本，存储类型使用绿色 `success` 标签，并在不同列中展示。项目组、项目及其他展示存储集群的列表和详情不得再把名称与类型组合在同一单元格，也不得只显示其中一项。

## 范围

- 用户目录列表：保留现有两列，改为复用共享存储类型标签。
- 项目组列表、项目详情：把组合列拆为“存储集群”和“存储类型”。
- 项目列表：把“集群类型”改为“存储集群”和“存储类型”，支持一个项目关联多个集群。
- 容量池、存储空间、Qtree（NetApp）列表：在现有存储集群列旁新增存储类型列。
- 存储集群管理列表：新增存储类型列。
- 表单、筛选器和只描述设备类型的业务文案不属于本次表格展示统一范围。

## 方案

### 共享组件

新增 `StorageTypeTag.vue`：

- 有值时渲染 `ElTag type="success"`。
- 空值时渲染普通文本 `-`。
- 页面不再自行决定存储类型颜色。

### 项目列表接口

`ProjectOverview` 保留 `storage_cluster_types`，新增 `storage_clusters`，元素包含 `id`、`name`、`storage_type`。项目列表继续先分页获取当前页项目，再对当前页项目 ID 批量查询项目组关联的存储集群：

- 按存储集群 ID 去重。
- 按集群名称、ID 稳定排序。
- 不加载完整项目组集合，不新增数据库迁移。

### 多集群展示

项目列表的两个单元格都按 `storage_clusters` 的同一顺序分行展示。名称列显示普通文本，类型列逐项使用共享标签，保证名称与类型位置对应。无关联集群时两列都显示 `-`。

### 响应式规则

保持现有视口策略，不增加窄屏必显列：

- 项目列表的两个集群列沿用原“集群类型”次要列显隐规则。
- 项目组、容量池、存储空间、Qtree 的存储类型列与对应存储集群列使用相同显隐条件。
- 存储集群管理页的类型列作为次要列，在窄屏隐藏。

## 测试

- 后端：项目概览 schema 包含集群摘要；批量汇总覆盖去重、排序、空项目和原兼容字段。
- 前端：共享标签覆盖有值和空值；静态全站契约锁定独立“存储集群/存储类型”列并禁止组合展示；项目列表覆盖多集群与空状态。
- 验证：聚焦测试、受影响文件 ESLint、前端全量测试与覆盖率、后端聚焦测试、生产构建、`git diff --check`。

## 风险与边界

- 项目列表响应新增只读摘要字段，旧客户端继续使用 `storage_cluster_types` 不受影响。
- 多集群项目会增加单行高度，但不引入横向滚动或额外交互。
- 浏览器视觉验收需要有效登录态和实际包含多集群项目的数据。
