# 导航顺序与图标优化设计

## 目标

使 DiskPulse 管理后台的侧栏符合常见运维系统的阅读顺序，并让每个可见入口具备清晰、语义一致的 Remix Icon。

## 范围与约束

- 一级顺序保持为：概览、用户目录、项目、项目组、AI 助手、告警、系统管理。
- 系统管理子项顺序调整为：存储集群、容量池、存储空间、Qtree（NetApp）、项目组标签、用户信息管理、系统设置、AI 中心。
- 图标由路由 `meta.icon` 声明并由既有 `RouteMenuItem` 渲染；不修改菜单组件层级、路由路径、权限、可见性或后端接口。
- 继续保留隐藏的离职备份及详情路由，不能因本次显示优化删除。

## 方案选择

只替换图标无法改善系统管理的认知顺序；引入三级分类会超过当前两级菜单组件的边界。采用调整已有子路由声明顺序、补齐 `meta.icon` 的最小方案。

## 验证

新增路由菜单单测，断言系统管理可见子项顺序与关键图标；运行聚焦路由测试、目标 ESLint 与生产构建。真实登录态下的权限过滤由既有逻辑负责，本轮不改变其实现。
