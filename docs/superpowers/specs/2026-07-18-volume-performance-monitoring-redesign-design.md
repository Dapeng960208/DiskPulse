# 存储空间性能监控重设计

## 目标

将存储空间详情从“容量趋势加重复告警”调整为“容量变化加关联性能分析”。存储空间与项目目录按一对一绑定处理，容量告警只由项目目录侧负责，存储空间不再生成、查询或展示重复告警。

## 已确认事实与设计假设

- 现有 `/admin/volume/:id` 使用通用 `RealTimePage`，会调用 `/volumes/{id}/realtime` 并额外查询 `related_type=Volume` 的告警。
- 容量历史存放于 QuestDB 的 `volume_storage_usages`；性能历史存放于 `storage_performance_metrics`，已有读/写/总延迟、IOPS、吞吐量字段。
- 当前数据模型通过 `Group.volume_id` 或 `Group.qtree_id` 关联存储目标，但没有数据库唯一约束保证存储空间与项目目录一对一。
- 本设计将“项目目录”定义为绑定 `volume_id` 的 `Group`，并要求每个存储空间最多绑定一个项目目录；无绑定或历史重复绑定记录不生成容量告警，页面明确显示关联状态。

## 方案比较

1. 扩展通用 `RealTimePage`：改动少，但会把存储空间的性能和告警差异继续耦合进 Aggregate、Qtree、项目和用户目录页面。
2. 新建专用 `VolumeMonitoringPage`（推荐）：保留其他资源的通用实时页，只让存储空间详情拥有容量与性能双区域、绑定信息和无告警语义。
3. 跳转到存储集群性能页：复用最多，但用户会失去与当前存储空间容量趋势并排分析的上下文。

## 页面设计

### 查询栏

- 第一行保留“存储空间名”和“时间范围”。
- 将单选“指标”替换为“性能指标”多选下拉框，默认选中“总延迟、IOPS、吞吐量”；指标选项为读延迟、写延迟、总延迟、IOPS、吞吐量。
- 保留“重置、搜索”，搜索一次性刷新容量和所有已选性能指标；时间变更遵循同一刷新语义。

### 关联信息

- 移除现有“存储空间名、限额、使用量、利用率”描述表，避免把静态容量摘要占据主工作区。
- 在标题副文案下显示紧凑的只读关联标记：`关联项目目录：<项目 / 目录名>`、`所属存储集群：<集群名>`；未绑定时显示“未关联项目目录”，并禁用性能曲线区域的请求。

### 图表工作区

- 固定第一张“存储空间容量变化”曲线图，展示已用容量；没有多个容量口径的选择器，也不显示阈值线或容量告警。
- 每个选中的性能指标生成一张独立曲线图，而不是在一个不同量纲的图内叠线。各图共享时间轴，独立 y 轴单位和 tooltip：毫秒、IOPS、B/s。
- 性能数据由关联存储集群的性能遥测按当前存储空间稳定对象标识解析；缺少稳定标识或采集时无法关联时不做名称猜测，在对应图内显示“该时间范围暂无性能数据”，不让其他图失败。
- 页面采用响应式两列图表网格；容量图占首行全宽，性能图在宽屏两列、窄屏一列，图表标题统一位于卡片顶部。

### 告警边界

- 删除 Volume 详情页的告警面板、告警请求和阈值状态读取。
- 取消 `related_type=Volume` 的容量告警生成、去重、通知和管理入口；保留既有历史记录只读归档，列表中不再把它们作为活跃告警返回。
- 项目目录（`Group`）是容量告警的唯一归属；同一容量事实只在其关联项目目录上评估一次，告警详情包含存储空间名称作为定位信息。

## 后端与数据契约

- 新增“存储空间监控详情”聚合接口，返回关联项目目录、存储集群、容量序列和按指标分组的性能序列；接口不返回 Volume 告警。
- 请求参数为 `start_time`、`end_time` 和 `metrics[]`；后端仅接受允许列表中的指标，默认三项性能指标，最大五项。
- 在绑定写入路径增加一对一校验和数据库唯一约束（仅对 `Group.volume_id` 的非空值生效）；迁移前先报告并人工处理重复绑定，不能静默选择其中一个目录。
- 采集链路持久化厂商稳定对象 ID，并建立 Volume 到性能对象的映射；集群分析和卷详情均只消费该映射，响应通过 `match_source=stable_id|unmatched` 标明状态。

## 错误处理与降级

- 容量数据失败时仅容量图显示可恢复错误；性能图仍可独立加载。
- 没有关联项目目录、没有性能映射或指标不受厂商支持时，返回结构化空状态与原因码，前端展示清晰空态。
- 无权限访问关联项目目录或集群时返回现有授权错误，不以名称或性能数据绕过项目隔离。

## 测试与验收

- 前端先编写失败测试，覆盖：不渲染 Volume 告警、默认三张性能图、指标多选增删图表、单图空态、时间范围同步和小屏单列布局。
- 后端先编写失败测试，覆盖：一对一绑定冲突、Volume 不触发告警、Group 只产生一条容量告警、性能指标白名单、稳定 ID 优先和名称回退范围限制。
- 迁移测试覆盖历史重复绑定阻断及约束生效；QuestDB 查询测试覆盖容量与性能时间窗、按指标分组和无数据返回。
- 验收时检查 `/admin/volume/:id` 不再发送 Volume 告警请求，容量图及所选性能图正确显示，并用真实 NetApp / Isilon 手工集成检查验证对象映射。

## 风险与取舍

- 强制一对一前必须清理历史重复绑定；这是数据治理前置条件，不能通过 UI 隐藏掩盖。
- 不同厂商的性能对象标识可能不一致，因此稳定 ID 映射是长期正确性边界；本版本选择未匹配置空，避免名称匹配把其他对象的指标错误关联到卷。
- 此设计只移除存储空间容量告警，不移除项目目录、存储集群设备事件或独立性能异常事件。
