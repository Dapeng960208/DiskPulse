# 项目组额度与实时趋势修复设计

## 范围

- 在“项目详情 → 项目组”页签展示硬限额、软限额、使用量、硬限额使用率和软限额使用率。
- 恢复项目组详情“容量趋势”页签中趋势图与关联告警工作区的可见高度。
- 保留项目组列表、项目内项目组列表和项目组详情中已删除的存储目标类型前缀，只展示存储目标名称。

## 设计

项目组表格复用“项目详情 → 用户目录”现有的额度展示规则：容量优先读取 `capacity` 字段级单位，缺少硬限额或软限额时显示明确状态，使用率由同一 `Progress` 组件按原始 `used` 与对应限额计算。接口与权限契约不变。

项目组详情继续复用 `RealTimePage`。为容量页签建立与用户目录详情一致的纵向 flex 高度链，使 `fillContent` 工作区得到非零剩余高度；不修改查询、图表或告警数据逻辑。

## 测试与成功标准

- 静态契约测试覆盖五个额度列、容量单位格式化、无软限额状态和两种进度条分母。
- 高度契约测试覆盖项目组容量页签的 flex 容器及 `RealTimePage` 填充模式。
- 聚焦 Vitest、相关页面契约测试、lint 和 `git diff --check` 通过。

## 风险与取舍

新增列会提高表格宽度，但 `DataTable` 已负责表格内部滚动，操作列继续固定在右侧。此次不扩展项目自身的软限额模型，也不修改后端响应。
