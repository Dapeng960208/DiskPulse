# 配额调整记录误显示为使用率告警修复设计

## 问题

配额调整记录复用 `storage_alerts.threshold` 和 `storage_alerts.avg_use_ratio` 保存旧、新硬限额。告警列表未区分 `alert_type`，把这类记录按普通告警格式化，导致容量值被显示成“使用率”。

## 方案

- 仅当 `alert_type=alert` 时生成“对象 + 事件 + 使用率”摘要并展示告警专属字段。
- `alert_type=quota_adjustment` 显示为“配额调整”，内容使用后端已保存的调整描述。
- 配额调整记录的事件类型、限额口径、级别、阈值和触发值显示 `-`，避免把复用字段解释为告警语义。
- 不修改数据库或历史记录，现有数据立即按正确语义展示。

## 验证

在现有告警前端契约测试中加入配额调整分流断言，先确认失败，再实现并运行同一聚焦测试；随后执行目标文件 ESLint。
