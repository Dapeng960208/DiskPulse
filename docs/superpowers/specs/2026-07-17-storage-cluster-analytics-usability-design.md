# 存储集群分析可用性改进设计

## 目标

修复存储集群详情页性能分析图的横轴标签与图表标题重叠问题；支持按多个存储对象筛选并对比性能数据；让详情页面包屑完整表达页面层级。

## 范围与边界

- 只修改前端和相关文档，不新增后端接口、数据库字段或权限。
- 性能对象筛选基于已请求到的 Top-N 性能结果本地执行；对象不在当前结果集时不会额外请求或显示。
- 详情页以声明式路由元数据指定父级面包屑；应用壳统一渲染，避免在单个业务页面硬编码。

## 方案

### 性能分析

- 在性能筛选栏中新增“对象”多选控件，选项来自当前性能数据的对象名称或对象标识。
- 未选择对象时保留全部结果；选择一个或多个对象时，柱状图类别、每个指标数据及表格行同步只保留选中对象。
- 图表组件接收统一的长标签布局配置：加大底部 grid 空间、标签按宽度截断并保留 tooltip，避免横轴标签进入图表标题区域。

### 面包屑

- 应用壳将路由匹配结果与详情路由的 `breadcrumb` 元数据组合为完整层级。
- 所有现有详情入口声明所属列表页作为父级；存储集群详情呈现“系统管理 / 存储集群 / 存储集群详情”。
- 普通列表页继续仅使用其现有路由标题，保持导航、权限与页面组件不变。

## 测试与验收

- 先扩展存储集群分析页面单测，验证对象多选影响图表类别、指标数据和表格数据，且清空选择恢复全部对象。
- 扩展应用壳/路由测试，验证系统管理下的详情路由渲染三级面包屑，并涵盖非管理详情的父级回填。
- 执行相关 Vitest 聚焦测试、目标 ESLint、生产构建，以及登录态浏览器视觉检查。

## 风险

- 本地对象筛选只在当前 Top-N 结果内比较；若用户需要跨全部对象排序或检索，需要后续单独设计后端查询参数和分页语义。
- 动态资源名称不用于全局面包屑，当前层级使用稳定的页面标题，避免数据尚未加载时出现抖动。
