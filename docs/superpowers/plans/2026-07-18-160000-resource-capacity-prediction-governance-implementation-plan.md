# 资源详情容量预测与 AI 治理实施计划

- 批准时间：2026-07-18 16:00 Asia/Shanghai
- 关联设计：[预测、RCA 与事件中心设计](../specs/2026-07-18-forecast-incident-center-design.md)

## 已批准范围

- 在项目组详情 `/group/:id` 和用户目录详情 `/usage/:id` 分别提供独立“容量预测”页签。
- 后台持续计算预测、AI 候选和评估；默认仅超级管理员可见。超级管理员全局开启用户可见性后，具有资源所属项目权限的角色才能读取。
- 保留 Theil-Sen 基线与现有 MAPE 口径；AI 预测只使用审批端点、受限聚合输入和严格的 P10/P50/P90 输出校验。失败或无效时回退基线。
- AI 候选必须在至少三个滚动 30 天留出窗口中平均 MAPE 优于基线至少 10%，且耗尽风险覆盖不变差，才允许超级管理员启用。

## 实施步骤

1. 以 RED 测试固定全局可见性、项目隔离、资源独立预测、计划约束、AI 输出回退与模型启用门槛。
2. 新增版本化候选预测、滚动评估、资源计划和单例设置的模型、迁移、CRUD、服务与受保护 API。
3. 为项目组和用户目录详情实现共享懒加载预测面板；增加超级管理员预测治理页面和全局发布开关。
4. 同步功能、RBAC、AI 数据边界、追踪和最新功能文档；运行聚焦后端与前端测试、迁移/类型检查及差异检查。

## 完成检查

- 非超级管理员在全局开关关闭时无法看到或读取预测；超级管理员可治理且后台任务语义不受开关影响。
- 项目组与用户目录预测不互相汇总；所有资源读写均服务端反查项目权限。
- 测试覆盖 AI 无效输出/超时回退、数据不足和过期、启用门槛、审计以及页签状态。
