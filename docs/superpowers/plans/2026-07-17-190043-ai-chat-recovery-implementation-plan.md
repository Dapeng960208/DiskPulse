# AI 对话可恢复机制实施计划

- 批准时间：2026-07-17 19:00:43（Asia/Shanghai）
- 设计依据：[AI 对话可恢复机制设计](../specs/2026-07-17-ai-chat-recovery-design.md)

## 已批准实施内容

1. 让 AI 客户端保留非法工具参数的安全调用上下文，并区分 JSON 格式错误和非对象参数。
2. 每轮最多两次把参数校验反馈交给模型修复；耗尽后完成降级回答，不显示泛化服务故障。
3. 工具轮次上限达到后禁用工具生成已有信息总结；失败时使用安全本地降级文案，并请求用户授权继续查询。
4. 通过既有审计载荷持久化 `degraded` 和 `recovery` 元数据；前端显示继续/重新查询操作。
5. 增加 SSE 终态和已知事件载荷检查，避免遗留生成中状态。
6. 同步 AI 对话文档、发布记录和可复现错误记录。

## 验证与完成检查

- 先添加后端与前端聚焦回归测试并确认 RED。
- 验证工具上限、两次参数修复、修复耗尽、本地降级、审计恢复元数据、历史重载、继续查询和 SSE 截断路径。
- 运行相关后端 pytest、前端 Vitest、静态检查与构建；如可用，执行登录态浏览器冒烟。

## 约束

- 保持既有 `ai.max_tool_iterations`、限流、只读工具和权限边界。
- 不新增数据库迁移。
- 不泄露原始非法工具参数、密钥、令牌或内部异常。
