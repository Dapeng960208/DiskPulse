# AI 工具功能白名单调整实施计划

- 批准时间：2026-07-17 20:28:41（Asia/Shanghai）
- 批准依据：用户在当前任务中明确要求关闭指定 AI 工具并开放超级管理员调整限额。

## 边界

- 只修改 AI 工具注册和执行权限，不删除现有页面、路由或直接 HTTP API。
- 关闭所有 AI 删除工具；关闭创建用户、创建容量池、创建存储空间、更新存储空间、全部 Qtree、系统设置更新和 AI 审计工具。
- 保持未列出的已授权工具不变。
- 对超级管理员显式开放项目组和用户目录的“调整限额”工具；普通用户不暴露且执行端继续拒绝。

## 实施与测试

1. 先新增聚焦测试，锁定关闭工具不在管理员注册表中、限额调整工具只在超级管理员注册表中。
2. 最小化调整路由 `openapi_extra` 元数据；不改变 `require_super_admin`、限流、Pydantic 请求体校验或内部 Bearer Token 继承。
3. 运行后端 AI 聚焦测试并同步专题文档、发布记录和错误复盘。

## 风险

- “关闭”按当前 AI 工具上下文解释；现有业务 API 和前端操作保持可用。
- 调整限额会触发真实存储设备写入，因此仅保留既有超级管理员和服务层校验，不绕过设备错误处理。
